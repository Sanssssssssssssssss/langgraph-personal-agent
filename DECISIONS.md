# DECISIONS

## D-001 CLI 优先
- 日期：2026-03-25
- 决策：第一版入口使用 CLI，而不是先做 FastAPI
- 原因：更快跑通状态图、工具路由和调试链路，适合阶段 0/1

## D-002 阶段边界固定为阶段 0 + 最小阶段 1
- 日期：2026-03-25
- 决策：当前不跨阶段扩展，不提前做 GraphRAG、本地推理优化、多 agent
- 原因：避免范围失控，遵循 PDF 推荐开发顺序

## D-003 存储边界使用 SQLite + Milvus Lite + 文件系统
- 日期：2026-03-25
- 决策：结构化数据进 SQLite，向量进 Milvus Lite，原始文件进 `data/uploads`
- 原因：职责清晰，便于逐步替换和调试

## D-004 文档主语言中文，代码与命名英文
- 日期：2026-03-25
- 决策：项目文档以中文为主，目录和代码标识使用英文
- 原因：兼顾沟通效率和工程可维护性

## D-005 初版使用规则驱动 intent 和本地哈希 embedding
- 日期：2026-03-25
- 决策：先不引入外部 LLM 或在线 embedding API
- 原因：确保最小系统可离线本地运行，先验证 graph、工具和存储骨架

## D-006 当前环境下向量层采用 Milvus 优先、文件索引回退
- 日期：2026-03-25
- 决策：保留 `MilvusLiteStore` 接口，但在当前 Python 3.13 环境中若 `milvus-lite` 不可安装，则自动回退到本地文件向量索引
- 原因：当前环境存在依赖分发限制，需要先保证骨架可运行，同时不改变后续回切 Milvus Lite 的接口边界
