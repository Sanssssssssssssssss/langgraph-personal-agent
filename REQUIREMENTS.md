# REQUIREMENTS

## 功能需求
1. 管理笔记：新增、查询、更新、删除
2. 管理提醒：新增、查询、完成、取消
3. 管理偏好：写入与读取用户偏好
4. 文件导入：接收文件、抽取文本、切分、索引、记录元数据
5. 知识检索：基于导入内容进行基础召回
6. 工具调用：由 LangGraph 显式路由到对应工具
7. 交互入口：第一版提供 CLI
8. 交互式终端：支持 REPL 会话模式和 slash 命令
9. 高风险写操作确认：对 destructive 操作先提示确认再执行
10. 检索 metadata/filter：导入文件后保留最小元数据，并支持按元数据过滤检索结果

## 非功能需求
- 状态显式可调试
- 模块可替换
- 数据持久化
- 结构简单稳健
- 每阶段有文档和验收标准
- 能在无外部模型 API 的情况下本地运行最小闭环
- 每次有效更新都要在 `logs/` 下新增一个 `txt` 更新记录

## 业务规则
- 不擅自扩展需求
- 任何实现都必须映射到本文件
- 高风险写操作后续需支持确认机制
- 当前阶段优先保证可运行、可排错、可扩展
- 在进入阶段 2 之前，先保证 VS Code 本地开发环境可一键启动
- 当前 destructive 确认范围仅包含 `note delete` 和 `remind cancel`
- 当前 retrieval filter 第一版仅覆盖 `file_id`、`source_name`、`extension`、`media_type`

## 验收标准
- 能完成最小多步任务
- 能写入和读取记忆
- 能导入文件并检索
- 能看到节点执行与错误来源
- 重启后 SQLite 数据仍可读取
- 交互式 CLI 能保留当前终端内的会话状态和待确认动作
- destructive 操作未确认前不得执行
- `retrieve` 能按 metadata 过滤结果，并在结果中回显过滤条件

## 缺失信息与当前假设
- 仓库名默认 `langgraph-personal-agent`
- 首批导入文件格式默认支持 `txt`、`md`、`pdf`
- 当前检索先使用本地哈希 embedding，后续再切换到更强 embedding 模型
